    v0.1.1 (June 2020)¶

    New functions

    1. Add the plotly sub-module, a set of Plotly functions comprising systole.plotly.plot_raw(), systole.plotly.plot_subspaces(), systole.plotly.plot_ectopic(), systole.plotly.plot_shortLong(), systole.plotly.plot_frequency(), systole.plotly.plot_nonlinear(), systole.plotly.plot_timedomain().

    2. Add plotly.utils.simulate_rr(), for random RR interval simulation with different kind of artefacts. Can also return peak vector.

    3. The correction sub-module has been largely rewritten and now include systole.correction.correct_extra(), systole.correction.correct_missed(), systole.correction.interpolate_bads(), systole.correction.correct_rr(), systole.correction.correct_peaks(), systole.correction.correct_missed_peaks(), systole.correction.correct_extra_peaks(). These function can correct artefacts either using peaks addition/removal or by interpolation of the RR time series.

    Enhancements

    1. The detection sub-module has been improved. It is now about 10x faster and return more information. The main function has been renamed to systole.detection.rr_artefacts().

    Bugfixes

    1. systole.correction.interpolate_clipping(): add exception in case of clipping artefacts at the edge of the signal segment. This can cause cash during recording. The default behavior is now to decrement the last/first item in case of threshold value. The threshold can be changed manually. This procedure can result in slightly inaccurate interpolation, using a longer recording should always be preferred when possible.

    2. The PPG signal simulator used for testing can now run infinitely.

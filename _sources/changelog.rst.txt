v0.1.1 (June 2020)
------------------

**New functions**

a. Add the `plotly` sub-module, a set of Plotly functions comprising `plot_raw`, `plot_subspaces`, `plot_ectopic`, `plot_shortLong`, `plot_frequency`, `plot_nonlinear`, `plot_timedomain`.
b. Add `simulate_rr()`, for random RR interval simulation with different kind of artefacts. Can also return peak vector.
c. The `correction` sub-module has been largely rewritten and now include `correct_extra`, `correct_missed`, `interpolate_bads`, `correct_rr`, `correct_peaks`, `correct_missed_peaks`, `correct_extra_peaks`. These function can correct artefacts either using peaks addition/removal or by interpolation of the RR time series.

**Enhancements**

a. The `detection` sub-module has been improved. It is now about 10x faster and return more information. The main function has been renamed to `rr_artefacts`.

**Bugfixes**

a. `interpolate_clipping`: add exception in case of clipping artefacts at the edge of the signal segment. This can cause cash during recording. The default behavior is now to decrement the last/first item in case of threshold value. The threshold can be changed manually. This procedure can result in slightly inaccurate interpolation, using a longer recording should always be preferred when possible.
b. The PPG signal simulator used for testing can now run infinitely.
